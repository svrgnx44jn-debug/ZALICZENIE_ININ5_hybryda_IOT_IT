# CONTRIBUTING

Dziękujemy za zainteresowanie współpracą przy projekcie „Przewodnik po stolicach”.

Poniżej znajdują się zasady, których należy przestrzegać podczas pracy
nad repozytorium.

## Zasady współpracy
- Każda nowa funkcjonalność lub zmiana powinna być wykonywana na osobnej gałęzi
- Zabronione jest bezpośrednie wprowadzanie zmian do gałęzi `main`
- Zmiany należy zgłaszać poprzez Pull Request

## Nazewnictwo branchy
Gałęzie powinny być nazywane według schematu: stolica-imię_nazwisko

## Zasady tworzenia commitów
- Commity powinny być krótkie i czytelne
- Wiadomość commita powinna opisywać wykonaną zmianę
- Zalecany język: polski

Przykłady:
- `Dodanie sekcji Bangkok`
- `Dodanie zdjęcia Bangkoku`
- `Poprawa opisu stolicy`

## Pull Request
- Pull Request należy tworzyć z własnej gałęzi do `main`
- W opisie PR należy jasno napisać, co zostało dodane lub zmienione
- Pull Request musi zostać zatwierdzony przez prowadzącego

## Komunikacja
- Komunikacja odbywa się poprzez:
  - komentarze w Pull Requestach
  - tablicę Trello przypisaną do projektu
- Wszelkie uwagi i sugestie należy zgłaszać w sposób kulturalny i rzeczowy

Dziękujemy za stosowanie się do powyższych zasad.
